Linguagem regular:

Aceita uma string que representa uma sequência de logs de acesso e ciclo de vida de um login, 
começando com LOGIN, seguido de AUTH, seguido de zero ou mais REQUEST, terminando com LOGOUT

Alfabeto: {LOGIN, AUTH, REQUEST, LOGOUT}

Pode ser representada por:
L1 = { LOGIN AUTH REQUEST* LOGOUT }
L1 = { w ∈ Σ* | w = LOGIN AUTH REQUESTk LOGOUT, k ≥ 0 }

Cadeias Aceitas:
LOGIN AUTH LOGOUT; LOGIN AUTH REQUEST LOGOUT; LOGIN AUTH REQUEST REQUEST LOGOUT

Cadeias Rejeitadas:
LOGIN AUTH REQUEST; LOGIN AUTH REQUEST REQUEST; LOGIN AUTH LOGOUT REQUEST LOGOUT

Linguagem livre de contexto:

Aceita uma string que contém blocos aninhados de transação, sendo que 
cada BEGIN precisa de um END correspondente, podendo estar aninhado

Alfabeto: {BEGIN, END}

Pode ser representada por:
V = { S } Σ = { BEGIN, END } R = { S → BEGIN S END | BEGIN END }

Cadeias Aceitas:
BEGIN END; BEGIN BEGIN END END; BEGIN BEGIN BEGIN END END END

Cadeias Rejeitadas:
BEGIN END END; BEGIN BEGIN END; BEGIN END BEGIN

Linguagem recursiva:

Aceita uma string que contém um trio balanceado de eventos OPEN, COMMIT e CLOSE,
com pelo menos 1 registro de cada

Alfabeto: {OPEN, COMMIT, CLOSE}

Pode ser representada por:
L3 = { OPEN^^n COMMIT^^n CLOSE^^n | n ≥ 1 }
L3 = { w ∈ Σ* | ∃ n ≥ 1 : w = OPEN^^n COMMIT^^n CLOSE^^n }

Cadeias Aceitas:
OPEN COMMIT CLOSE; OPEN OPEN COMMIT COMMIT CLOSE CLOSE; OPEN OPEN OPEN COMMIT COMMIT COMMIT CLOSE CLOSE CLOSE

Cadeias Rejeitadas:
OPEN OPEN COMMIT CLOSE CLOSE; OPEN COMMIT COMMIT CLOSE; OPEN OPEN OPEN COMMIT COMMIT CLOSE CLOSE CLOSE
